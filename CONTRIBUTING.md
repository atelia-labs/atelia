# Contributing to Atelia

Atelia is early, public, and moving quickly. Contributions are welcome through
issues, discussions, design notes, and pull requests.

## Where To Start

- Use issues for bugs, concrete feature requests, and scoped work.
- Use discussions for open-ended questions, proposals, and product direction.
- Use pull requests for changes that are ready to review.
- Project-wide philosophy, governance, and specifications belong in this repo.
- Implementation changes belong in the relevant implementation repo.

## Pull Requests

Keep pull requests focused. Include:

- the problem being solved;
- the changed files or docs;
- any test or review performed;
- follow-up work, if any.

For now, maintainers may merge small documentation-only pull requests after one
review. Larger specification or governance changes should link to an issue or
discussion.
